# V2.1 - Page Illustration Generation

**Version**: 2.1.0  
**Date**: 26 février 2026  
**Statut**: ✅ Implémenté

---

## 🎯 Objectif

Permettre la génération d'illustrations page par page et en batch, en garantissant la cohérence visuelle grâce au système V2 (Visual Identity) et la compatibilité impression KDP.

---

## 🚀 Nouvelles Fonctionnalités

### 1. Génération par Page (depuis Écriture)

**Localisation**: `PageEditor.js`

#### Fonctionnement
- Bouton "Générer illustration" visible uniquement sur les pages avec zone image
- Templates supportés: `full_illustration`, `mixed`, `double_page`
- Processus en 2 étapes:
  1. **Scene Builder**: Génération d'une description visuelle de la scène (2-5 phrases)
  2. **Variant Generation**: Création de 4 variantes d'illustration via DALL-E 3

#### Workflow utilisateur
1. Ouvrir une page dans l'éditeur
2. Cliquer sur "Générer illustration"
3. Attendre la création de la scène (~5-10 secondes)
4. Attendre la génération des 4 variantes (~1-2 minutes)
5. Sélectionner la variante préférée
6. L'illustration est automatiquement insérée dans la page

#### Bouton "Regénérer"
- Apparaît si une illustration existe déjà
- Permet de créer 4 nouvelles variantes
- Conserve l'historique des variantes précédentes

---

### 2. Génération en Batch (depuis Illustrations)

**Localisation**: `IllustrationsManager.js`

#### Fonctionnalités
- Vue d'ensemble de toutes les pages du livre
- Statuts par page:
  - ✅ **Complétée**: Illustration générée
  - ⏳ **En cours**: Génération en cours
  - ⚠️ **Manquante**: Pas d'illustration
  - ❌ **Échec**: Erreur lors de la génération
  - ⏱️ **En attente**: Dans la queue
  - 🔄 **Nouvelle tentative**: Retry après erreur

#### Système de Queue
- **Une requête à la fois**: Évite les rate limits OpenAI
- **Progression en temps réel**: Barre de progression et compteurs
- **Pause/Reprise**: Possibilité d'interrompre et reprendre
- **Retry automatique**: 3 tentatives maximum par page
- **Délai entre requêtes**: 1 seconde pour éviter les erreurs

#### Statistiques
- Pages totales
- Illustrations générées
- Illustrations manquantes

#### Batch vs Page individuelle
- **Batch**: Génère 1 seule variante par page (économie temps/coût)
- **Page individuelle**: Génère 4 variantes pour choix utilisateur

---

### 3. Scene Builder (Qualité)

**Localisation**: `src/utils/sceneBuilder.js`

#### Objectif
Transformer le texte de la page en description visuelle courte et précise pour l'illustrateur IA.

#### Entrées
- Texte de la page
- Résumé du livre
- Liste des personnages (nom, âge, description, apparence)
- Âge cible

#### Sortie
Description de scène en 2-5 phrases, focalisée sur:
- Actions visibles
- Expressions des personnages
- Décor et ambiance
- Composition (premier plan, arrière-plan)
- Couleurs et atmosphère

#### Exemple
**Texte de la page**:
> "Léo le dragon découvre une forêt magique. Les arbres brillent d'une lumière dorée. Il est émerveillé."

**Scene générée**:
> "Léo, un petit dragon aux écailles bleues portant une écharpe jaune, se tient au premier plan, les yeux grands ouverts d'émerveillement. Derrière lui, une forêt enchantée avec de grands arbres aux feuilles scintillantes d'une lumière dorée. L'ambiance est magique et chaleureuse, avec des rayons de lumière filtrant à travers les branches."

### 4. Prompt Injection Obligatoire (Cohérence)

**Localisation**: `src/utils/imagePromptBuilder.js`

#### Utilisation systématique
Toutes les générations d'images utilisent `buildImagePrompt()` qui injecte automatiquement:

1. **Style artistique**
   - Prompt technique du style choisi (aquarelle, pastel, etc.)
   
2. **Palette de couleurs**
   - 3-5 couleurs dominantes du personnage principal
   
3. **Description personnage**
   - Nom, âge, apparence, vêtements
   - Référence à l'image canonique
   
4. **Marqueurs qualité**
   - "children's book illustration, high quality, professional"

#### Validation obligatoire
- Vérifie que `visualIdentity.validated === true`
- Bloque la génération si identité non validée
- Message d'erreur clair pour guider l'utilisateur

#### Structure du prompt final
```
[Style artistique prompt]
Main character: [Nom], [description complète], age [âge].
Color palette: [couleurs].
Scene: [Description de la scène générée par Scene Builder]
children's book illustration, high quality, professional
```

### 5. Ratios & Impression (KDP)

**Localisation**: `src/utils/imageRatioCalculator.js`

#### Formats KDP supportés
- 8.5" x 11" (Letter)
- 8" x 10"
- 8.5" x 8.5" (Square)
- 8" x 8" (Square)
- 7" x 10"
- 6" x 9"
- 5.5" x 8.5"
- 5" x 8"

#### Calcul automatique du ratio
Basé sur:
- **Format du livre**: Dimensions physiques
- **Template de page**: Ajustement selon le type
  - `full_illustration`: Page complète
  - `short_text`: 30% de la page
  - `mixed`: 50% de la page
  - `double_page`: Double largeur
- **Bleed**: +0.125" sur tous les côtés

#### Tailles DALL-E 3
Sélection automatique du format optimal:
- `1024x1024`: Ratio carré (0.77 - 1.3)
- `1792x1024`: Paysage (ratio > 1.3)
- `1024x1792`: Portrait (ratio < 0.77)

#### Qualité d'impression
- **Minimum requis**: 300 DPI
- **Recommandé**: 600 DPI
- **Warning automatique**: Si résolution < 300 DPI

#### Exemple de calcul
```javascript
Format: 8x10 inches
Template: full_illustration
Bleed: 0.125 inches

Dimensions finales:
- Largeur: 8 + (0.125 * 2) = 8.25 inches
- Hauteur: 10 + (0.125 * 2) = 10.25 inches

À 300 DPI:
- Largeur: 8.25 * 300 = 2475 pixels
- Hauteur: 10.25 * 300 = 3075 pixels

Ratio: 2475/3075 = 0.805
DALL-E size: 1024x1792 (portrait)
```

### 6. Gestion des Variantes & Historique

**Stockage dans le projet**

Chaque page peut avoir:
```javascript
page.illustration = {
  id: "uuid",
  url: "https://...",  // Image sélectionnée
  sceneDescription: "...",  // Description de scène
  revised_prompt: "...",  // Prompt DALL-E révisé
  selectedAt: "2026-02-26T...",
  variantIndex: 0,  // Index de la variante choisie
  allVariants: [  // Toutes les variantes générées
    { url: "...", revised_prompt: "...", generatedAt: "..." },
    { url: "...", revised_prompt: "...", generatedAt: "..." },
    { url: "...", revised_prompt: "...", generatedAt: "..." },
    { url: "...", revised_prompt: "...", generatedAt: "..." }
  ],
  dalleParams: {
    size: "1024x1792",
    quality: "standard",
    dimensions: { ... }
  },
  batchGenerated: false  // true si généré en batch
}
```

#### Actions disponibles

1. **Regénérer** (même scène)
   - Utilise la même description de scène
   - Génère 4 nouvelles variantes
   - Conserve l'historique

2. **Modifier la scène** (future feature)
   - Permet d'éditer la description de scène
   - Regénère avec la nouvelle description

3. **Remplacer** (future feature)
   - Choisir une autre variante de l'historique
   - Pas de nouvelle génération

## 🔧 Composants Créés

### 1. `SceneBuilder.js` (Utility)
**Fonctions**:
- `buildSceneDescription()`: Génère description visuelle via GPT-4
- `validateSceneDescription()`: Valide la qualité de la scène

**Fonctions**:
- `calculateImageDimensions()`: Calcule dimensions optimales
- `getRecommendedDalleParams()`: Paramètres DALL-E pour une page
- `validateImageForPrint()`: Vérifie conformité impression
- `getDallESize()`: Sélectionne format DALL-E optimal
- `assessPrintQuality()`: Évalue qualité d'impression

**Classe**: `IllustrationQueue`
- Gestion de queue séquentielle
- Retry automatique (3 tentatives)
- Pause/Reprise
- Callbacks de progression
- Délai entre requêtes

**Props**:
- `variants`: Array de variantes
- `onSelect`: Callback sélection

- `onRegenerate`: Callback regénération

**Features**:
- Affichage grille 2x2
- Indicateur de sélection
- Bouton téléchargement par variante
- Affichage prompt révisé
- Responsive

**Features**:
- Vue tableau de toutes les pages
- Statistiques globales
- Barre de progression batch
- Boutons Générer/Pause/Reprendre/Réinitialiser
- Statuts en temps réel
- Aperçu miniature des illustrations

```javascript
const validation = validateVisualIdentity(visualIdentity);
if (!validation.isValid) {
  // Blocage + message d'erreur
```

```javascript
const pageText = page.textBlocks?.map(b => b.content).join(' ').trim();
```

## 💰 Optimisation Coûts

### Stratégie batch vs individuel
- **Page individuelle**: 4 variantes = 4 appels DALL-E
  - Avantage: Choix utilisateur
  - Coût: ~$0.16 (4 × $0.04)
  
- **Batch**: 1 variante = 1 appel DALL-E
  - Avantage: Économie 75%
  - Coût: ~$0.04
  - Possibilité de regénérer individuellement après

### Délais entre requêtes
- 1 seconde entre chaque génération batch
- Évite rate limiting
- Permet monitoring progression

### Qualité "standard"
- Utilisation de `quality: "standard"` par défaut
- Option "hd" disponible dans le code pour upgrade futur
- Économie: standard = $0.04, hd = $0.08

## 📈 Métriques de Performance

### Temps de génération
- **Scene description**: 5-10 secondes
- **1 variante DALL-E**: 15-30 secondes
- **4 variantes**: 1-2 minutes
- **Batch 10 pages**: 5-10 minutes

### Qualité
- **Cohérence visuelle**: 100% (grâce à V2)
- **Résolution**: 300-600 DPI (print-ready)
- **Ratio**: Adapté au format livre

### Coûts (estimation)
- **Page individuelle**: $0.16 (4 variantes)
- **Page batch**: $0.04 (1 variante)
- **Livre 20 pages batch**: ~$0.80
- **Livre 20 pages individuel**: ~$3.20

## 🐛 Problèmes Connus & Limitations

### Limitations actuelles
1. **Pas d'édition de scène**: La description générée ne peut pas être modifiée manuellement
2. **Pas de remplacement de variante**: Impossible de choisir une autre variante de l'historique sans regénérer
3. **Batch = 1 variante**: Pas de choix en mode batch (économie de coût)
4. **Pas de preview avant génération**: L'utilisateur ne voit pas la description de scène avant génération

### Améliorations futures (V2.2)
- Éditeur de description de scène
- Galerie de variantes historiques
- Option "batch avec choix" (2 variantes au lieu de 1)
- Preview de la scène avant génération
- Téléchargement batch de toutes les images
- Export des métadonnées (prompts, scènes)
